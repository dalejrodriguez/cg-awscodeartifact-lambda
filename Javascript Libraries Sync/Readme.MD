# Chainguard вҶ’ AWS CodeArtifact NPM Sync

Automated Lambda functions that mirror NPM packages from [Chainguard Libraries](https://edu.chainguard.dev/chainguard/libraries/javascript/overview/) to AWS CodeArtifact. Provides supply-chain security by pulling vetted packages from Chainguard instead of public npm.

## Overview

This project deploys two Lambda functions and supporting infrastructure via AWS SAM:

| Function | Handler | Purpose |
|---|---|---|
| `chainguard-codeartifact-sync` | `lambda_function.lambda_handler` | Sync **exact lockfile versions** вҖ” mirrors only the specific versions your project uses |
| `chainguard-codeartifact-sync-all` | `sync_all_versions.lambda_handler` | Sync **all available versions** вҖ” fully populates your CodeArtifact mirror for every package in the lockfile |

Both functions share the same architecture: two-phase checkpointing, automatic continuation across Lambda invocations, error tracking, and garbage cleanup.

### Two-Phase Checkpointing

Large lockfiles (50,000+ packages) cannot be processed in a single 15-minute Lambda invocation. The function splits work into two checkpointable phases:

**Phase 1 вҖ” Discovery:** Iterates package names, fetches metadata from Chainguard, diffs against CodeArtifact, and builds a work items list. Checks remaining execution time before each package and saves a checkpoint if running low.

**Phase 2 вҖ” Processing:** Iterates work items, downloads tarballs from Chainguard, and publishes to CodeArtifact via the npm PUT protocol. Same time-check and checkpoint behavior.

```
Invocation 1: Discovery [packages 1-800]     вҶ’ checkpoint вҶ’ self-invoke
Invocation 2: Discovery [packages 801-1500]   вҶ’ checkpoint вҶ’ self-invoke
Invocation 3: Discovery [1501-2000] complete  вҶ’ Processing [items 1-400] вҶ’ checkpoint
Invocation 4: Processing [items 401-900]      вҶ’ complete
```

Checkpoints are saved to S3 and include the full state: work items, counters, error list, and progress indices. On successful completion, checkpoint files are automatically deleted.

## Prerequisites

- AWS CLI and SAM CLI installed
- Chainguard Libraries credentials (`CGR_USERNAME` and `CGR_TOKEN`)
- An AWS account with permissions to create Lambda, S3, CodeArtifact, and IAM resources
